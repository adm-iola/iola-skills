# Проверка фактов по открытым данным

Используй этот skill для любых ответов, где важна точность: ФИО руководителей,
адреса, телефоны, сайты, номера организаций, списки учреждений и статистика.

## Источник истины

Основной источник истины - MCP:

```text
https://apiiola.yasg.ru/mcp
```

Допустимый fallback для клиентов без MCP - публичный REST API:

```text
https://apiiola.yasg.ru/api/v1
```

## Запреты

- Не отвечай по памяти, если вопрос можно проверить через MCP.
- Не исправляй "очевидные" опечатки в данных без источника.
- Не смешивай факты из разных записей.
- Не выдавай вероятностный ответ как подтвержденный факт.

## Формат поведения

1. Найди слой через `layer_suggest`, если слой не очевиден.
2. Получи контекст через `layer_answer_context`.
3. Сверь ответ с `facts`, `results` или `recommended_answer_ru`.
4. Если данных нет, скажи: "В доступных публичных данных это не найдено."

Если пользователь просит проверить ответ модели, явно отдели:

- что подтверждается данными;
- что не найдено;
- что требует ручной проверки источника.
